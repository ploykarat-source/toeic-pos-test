# คู่มือการติดตั้งระบบขึ้น Cloudflare Pages & D1 Database (ผ่าน Dashboard UI)

เนื่องจากสถาปัตยกรรมของแอปพลิเคชันนี้ถูกออกแบบมาในรูปแบบ **Serverless Full-stack SPA** (ไม่มี Build step) คุณจึงสามารถติดตั้งและเชื่อมโยงระบบเข้ากับฐานข้อมูล Cloudflare D1 ได้ฟรี ผ่านหน้าเว็บ Cloudflare Dashboard โดยตรง โดยไม่ต้องติดตั้งโปรแกรมสำหรับพัฒนาหรือ Node.js ในเครื่องคอมพิวเตอร์ของคุณ

ทำตาม 4 ขั้นตอนด้านล่างนี้เพื่อนำระบบขึ้นออนไลน์:

---

## 📌 สิ่งที่ต้องเตรียมก่อนเริ่มต้น
1. บัญชีผู้ใช้ **Cloudflare Account** (หากยังไม่มี สามารถสมัครฟรีได้ที่ [dash.cloudflare.com](https://dash.cloudflare.com))
2. นำไฟล์โปรเจกต์ทั้งหมดนี้ขึ้นไปเก็บบน **GitHub** หรือ **GitLab Repository** ของคุณ (เช่น สร้าง Repository ชื่อ `toeic-pos-test` และ Push โฟลเดอร์ที่มีไฟล์ `index.html`, `app.js`, `functions/` ฯลฯ ขึ้นไป)

---

## 🛠️ ขั้นตอนที่ 1: สร้างและกรอกฐานข้อมูล D1 บน Cloudflare
Cloudflare D1 คือฐานข้อมูล SQL (SQLite) บนระบบคลาวด์ที่เราจะใช้บันทึกประวัติคะแนนสอบ

1. เข้าสู่ระบบ **Cloudflare Dashboard**
2. เมนูด้านซ้ายคลิกที่ **Workers & Pages** -> เลือกย่อย **D1**
3. คลิกปุ่ม **Create** (หรือ **Create database**)
4. เลือก **Dashboard** (สร้างผ่านหน้าเว็บ)
5. ตั้งชื่อฐานข้อมูล: **`toeic-pos-db`** จากนั้นกดปุ่ม **Create**
6. เมื่อสร้างเสร็จแล้ว จะเข้ามาในหน้าของฐานข้อมูลนั้น:
   * คลิกแท็บ **Console** ด้านบนสุด
   * เปิดดูไฟล์ **`schema.sql`** ในโฟลเดอร์โครงการของคุณ แล้วคัดลอก (Copy) คำสั่ง SQL ทั้งหมดมาวางในช่อง Console
   * คลิกปุ่ม **Execute** เพื่อเริ่มสร้างตาราง `users` และ `progress` ในฐานข้อมูล

---

## 🚀 ขั้นตอนที่ 2: ติดตั้ง Web Application ขึ้น Cloudflare Pages
เราจะเชื่อมระบบเข้ากับ Git เมื่อคุณอัปเดตโค้ดใน GitHub ในอนาคต หน้าเว็บจะอัปเดตให้เองอัตโนมัติ

1. ที่เมนูด้านซ้ายของ Cloudflare คลิกที่ **Workers & Pages** -> เลือกแท็บ **Pages**
2. คลิกปุ่ม **Connect to Git**
3. เลือกบัญชี GitHub ของคุณและเลือก Repository ที่คุณอัปโหลดไฟล์โปรเจกต์นี้ไว้
4. กด **Begin setup**
5. ตั้งค่าการ Build (กรอกตามข้อมูลนี้):
   * **Project name:** ตั้งชื่อเว็บของคุณ (เช่น `toeic-pos-mastery`)
   * **Production branch:** `main` (หรือตามสาขาที่คุณใช้ใน Git)
   * **Framework preset:** เลือกเป็น **`None`** (เนื่องจากเราเขียน HTML/JS/CSS แท้ ไม่ต้องคอมไพล์)
   * **Build command:** ปล่อยว่างไว้ไม่ต้องกรอก
   * **Build output directory:** ปล่อยว่างไว้ (หรือใส่จุด `.` หรือ `./` เพื่อให้ใช้ Root Directory)
6. คลิกปุ่ม **Save and Deploy**
7. รอระบบทำการติดตั้งสักครู่ หน้าเว็บของคุณจะถูกนำขึ้นออนไลน์ โดยมีลิงก์ทดสอบเช่น `https://project-name.pages.dev` (ในขั้นตอนนี้หน้าเว็บจะรันได้ แต่ระบบสมัครสมาชิก/เข้าสู่ระบบจะยังขึ้น Error เพราะเรายังไม่ได้เชื่อมฐานข้อมูล D1 ในขั้นตอนถัดไป)

---

## 🔗 ขั้นตอนที่ 3: เชื่อมต่อ (Bind) ฐานข้อมูล D1 เข้ากับ Pages
ขั้นตอนนี้เป็นการระบุให้ APIs หลังบ้านของ Pages (ที่อยู่ในโฟลเดอร์ `functions/api/`) รู้จักฐานข้อมูล D1

1. ในหน้าโปรเจกต์ Pages ของคุณ (ที่เพิ่ง Deploy เสร็จในขั้นตอนที่ 2)
2. คลิกที่แท็บ **Settings** ด้านบนสุด
3. เมนูด้านซ้ายเลือกหัวข้อ **Functions**
4. เลื่อนลงมาด้านล่างสุดจะเจอหัวข้อ **D1 database bindings**
5. คลิกปุ่ม **Add binding**
6. กรอกรายละเอียดดังนี้:
   * **Variable name (ชื่อตัวแปร):** ต้องพิมพ์เป็น **`DB`** ตัวใหญ่ทั้งหมดเท่านั้น (เพราะหลังบ้านเรียกใช้งานผ่านคำสั่ง `env.DB`)
   * **D1 database:** เลือกฐานข้อมูล **`toeic-pos-db`** ที่คุณสร้างไว้ในขั้นตอนที่ 1
7. คลิก **Save**
8. **สำคัญมาก:** ตรวจสอบว่าได้ตั้งค่า binding นี้ทั้งในส่วนของ **Production** และ **Preview** (หากมี) เพื่อให้ทุกสภาพแวดล้อมรันฐานข้อมูลตัวเดียวกันได้ถูกต้อง

---

## 🔄 ขั้นตอนที่ 4: สั่งติดตั้งใหม่อีกครั้ง (Re-deploy) เพื่อรับค่า Config
เพื่อให้โปรเจกต์ Pages นำตัวแปร `DB` ที่เรา Bind ไว้ไปเปิดใช้งานจริง เราต้องกดสั่ง Deploy ใหม่หนึ่งครั้ง

1. ในหน้าโปรเจกต์ Pages ของคุณ คลิกแท็บ **Deployments** ด้านบนสุด
2. จะเห็นรายการรันล่าสุดที่เป็นสีเขียว ให้คลิกที่จุดสามจุดด้านขวา
3. เลือก **Retry deployment** (หรือหากคุณทำการ Push โค้ดชิ้นเล็กๆ ขึ้น GitHub อีกรอบ ระบบก็จะ Auto-deploy ใหม่เช่นกัน)
4. รอจนแถบสีเขียวขึ้นเรียบร้อย

---

## 🎉 เสร็จสิ้นการติดตั้ง!
หลังจาก Re-deploy เสร็จแล้ว คุณสามารถเปิดใช้งานลิงก์เว็บ `https://your-project.pages.dev` ได้ทันที!
* หน้าล็อกอินจะแสดงผลขึ้นมา
* เมื่อผู้เรียนกรอก Username ระบบจะบันทึกข้อมูลและดึงความคืบหน้าขึ้น Cloudflare D1 Database
* ผู้เรียนสามารถเปิดเล่นบนคอมพิวเตอร์ มือถือ หรือเครื่องอื่นใดก็ได้ เพียงกรอก Username เดิม ก็จะสามารถโหลดคะแนนและระดับการเรียนย้อนหลังมาทำต่อได้ทันทีครับ!
